CUALQUIER agente que continúe el proyecto:
- DEBE releer este archivo antes de continuar
- DEBE reaplicar estas reglas como criterio permanente

Este archivo es la FUENTE ÚNICA DE VERDAD del proyecto.

================================================================
PRINCIPIO GENERAL DE RESPONSIVE DESIGN
================================================================

La página web debe ser completamente RESPONSIVE y adaptativa.

REGLA CLAVE:
La versión móvil NO debe ser una versión minimizada,
reducida o simplemente reacomodada del diseño de desktop.

Desktop, tablet y móvil deben considerarse
EXPERIENCIAS DIFERENTES, optimizadas para cada pantalla.

Cualquier cambio en desktop DEBE evaluarse
automáticamente en tablet y móvil, y viceversa.

================================================================
DISEÑO EN DISPOSITIVOS MÓVILES (PRIORIDAD ALTA)
================================================================

- El diseño móvil debe replantearse de forma independiente.
- El contenido debe organizarse principalmente en columna vertical.
- No se permite escalar visualmente el desktop hacia móvil.
- El hero móvil debe ser claro, directo y funcional.
- El usuario debe entender qué ofrece la página en menos de 3 segundos.
- No se deben usar animaciones pesadas en móvil.
- No se deben usar efectos que afecten rendimiento
  (parallax complejo, 3D pesado, video autoplay, partículas avanzadas).

================================================================
DESKTOP Y TABLET (LIBERTAD VISUAL CONTROLADA)
================================================================

- El diseño desktop puede ser más visual, amplio y expresivo.
- En desktop SÍ se permiten animaciones avanzadas y efectos visuales,
  siempre que no rompan la usabilidad ni la legibilidad.
- Se permite el uso de:
  • Animaciones complejas con CSS o JavaScript
  • Librerías como GSAP, Anime.js, Scroll-based animations
  • Partículas, fondos animados y efectos interactivos
  • Three.js y efectos 3D (uso responsable)
  • Transiciones elaboradas y microinteracciones
- Tablet debe ser un punto intermedio:
  • Puede heredar parte de los efectos de desktop
  • Pero debe priorizar rendimiento y claridad

REGLA:
Desktop es el espacio para impacto visual y diferenciación,
móvil es el espacio para claridad, velocidad y conversión.

================================================================
ICONOS FLOTANTES (WHATSAPP, TELÉFONO, TAWK.TO, ETC.)
================================================================

Si se utilizan iconos flotantes:

- Deben ser táctiles y accesibles en móvil.
- No deben cubrir contenido importante.
- No deben invadir formularios ni CTAs.
- Su tamaño y posición deben ajustarse según el viewport.
- En móvil deben ser visibles pero discretos.
- En desktop pueden tener mayor presencia visual.
- Deben respetar márgenes seguros (safe areas).

================================================================
TIPOGRAFÍA Y UNIDADES DE MEDIDA (OBLIGATORIO)
================================================================

TIPOGRAFÍA:
- El tipo de fuente PUEDE cambiar según el proyecto,
  siempre que cumpla criterios de legibilidad y rendimiento.
- Fuente recomendada por defecto: Inter (o equivalente).
- Fallback: system-ui, -apple-system, Segoe UI, sans-serif.
- Preferible una sola familia con distintos pesos.
- Pesos recomendados:
  • 400 → texto
  • 500–600 → subtítulos
  • 700 → títulos

LEGIBILIDAD:
- Texto siempre legible en todos los dispositivos.
- Tamaño mínimo en móvil:
  • Texto normal: 1rem (16px)
  • Títulos: 1.125rem (18px) o más
- Line-height:
  • Texto: 1.5
  • Títulos: 1.2

UNIDADES DE MEDIDA:
- rem → texto, padding, margin (prioritaria).
- em → componentes internos.
- % → layouts flexibles.
- vh / vw → uso controlado (hero).
- px → solo bordes, íconos y detalles finos.

PROHIBIDO:
- Texto base definido en px.
- Escalar visualmente desktop para móvil.

================================================================
BOTONES, CTAs Y FORMULARIOS
================================================================

- Botones fáciles de tocar en móvil.
- Un CTA principal claro por sección.
- Formularios optimizados para uso táctil.
- Evitar campos innecesarios en móvil.

================================================================
TIPOS DE PÁGINAS (APLICACIÓN UNIVERSAL)
================================================================

Estas reglas aplican a:
- Landing pages
- Sitios corporativos
- Portfolios
- Páginas personalizadas
- Páginas tipo ecommerce frontend
- Catálogos con contacto o pago vía WhatsApp

No existe excepción por tipo de proyecto.

================================================================
RENDIMIENTO
================================================================

- Priorizar carga rápida en todos los dispositivos.
- Imágenes optimizadas.
- En móvil, evitar librerías innecesarias.
- En desktop, los efectos deben aportar valor real,
  no solo decoración.

================================================================
REGLA FINAL (NO NEGOCIABLE)
================================================================

Si existe un conflicto entre:
- estética vs usabilidad móvil
- efecto visual vs rendimiento
- diseño desktop vs experiencia móvil

SIEMPRE se prioriza:
USABILIDAD + CLARIDAD + RESPONSIVE REAL
